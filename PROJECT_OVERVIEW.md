# OFT — Онлайн Фитнес Тренер

## Зачем проект
OFT — приложение для фитнес‑тренеров и их клиентов, которое помогает вести клиентов, выдавать план тренировок, показывать технику упражнений и отслеживать прогресс.

## Что делаем сейчас (этап MVP-Local)
- Без авторизации (все данные локально на устройстве).
- Без ИИ-агентов (пока заглушки/места под них).
- Без Telegram-бота (позже).
- Локальная база: SQLite (expo-sqlite).
- Основная цель этапа: собрать структуру, дизайн и логику экранов так, чтобы потом было легко добавить авторизацию и ИИ.

## Роли (пока условно)
- Тренер: видит список клиентов, может “добавить клиента” (локально), открыть профиль.
- Клиент: видит “главную”, карту мышц, план на сегодня, упражнения, прогресс.

## Основные экраны (MVP)
1) Старт/выбор режима (Тренер / Клиент) — временно вместо логина
2) Trainer Dashboard — список клиентов + создать клиента
3) Wizard добавления клиента — 3 шага (данные → цели/оборудование → черновик плана БЕЗ ИИ)
4) Client Home — “сегодня” + быстрый вход в карту мышц
5) Muscle Map — кликабельные мышцы → список упражнений
6) Exercise Catalog — фильтры и карточки
7) Exercise Detail — видео-заглушка/картинка, техника, подходы/повторы
8) Today Workout — список упражнений с чекбоксами выполнения
9) Progress — метрики и история (без реальных графиков на старте можно placeholder)

## Данные
Все хранится локально (SQLite). Нужны сиды: упражнения, группы мышц, 1–2 демо клиента, демо планы.

## Ограничения
- Никаких сетевых запросов.
- Никакой регистрации/логина.
- Фокус на UX, структуру и дальнейшую расширяемость.
